**推荐你把主项目包装成什么（最贴 PM 实习）**
- 产品定位：**“PM/商业研究 Copilot（竞品/行业/机会点/PRD 一条龙）”**
- 目标用户：产品经理/创业团队/咨询分析岗
- 核心卖点：**可追溯证据链（引用）、可控流程（HITL）、可接企业数据源（MCP）、可用内部知识（RAG）**

**升级路线（你做完就会同时体现 MCP + RAG + 多 Agent + 产品思维）**
1) **数据源层（MCP）**：做一个 “Connector Hub”
- 接入 2–3 个 MCP Server（例如 Notion/Google Drive/GitHub/Jira 任意组合），支持：列资源、拉文档、按关键词检索、权限与审计日志。
2) **知识层（RAG）**：把 `RAG_副本` 的能力变成 deep-search 的“企业知识检索工具”
- 用 MCP 把内部文档拉进来 → 分块/embedding → 向量库 → 检索 → 生成时强制引用（延续 deep-search 的 citation 机制）。
3) **多 Agent 升级**（在 `deep-search` 的 pipeline 上加“商业/产品角色”）
- 例：`Planner(研究计划)` → `Retriever(Web+RAG)` → `Business Analyst(市场/竞品/定价)` → `PM(需求拆解/PRD)` → `Critic(证据覆盖/风险与假设)`，并保留 HITL 审批点（计划/大纲/结论）。
4) **产品化闭环**
- 前端加：项目/Workspace、报告历史、导出（通过 MCP 写回 Notion/Docs）、“证据覆盖率/信息新鲜度”面板。
- 指标设计（写进简历/面试）：节省研究时间、一次通过率（少返工）、引用覆盖率、用户接受/导出率。
