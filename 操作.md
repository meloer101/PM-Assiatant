# 操作说明

## 一、PM Research Copilot（pm-copilot）— 前后端分离启动

项目为前后端分离：后端 ADK API 与前端 React 可**分别启动**，便于单独调试。

### 1. 安装依赖（首次或依赖变更时）

```bash
cd pm-copilot
make install
```

若未安装 `uv`，Makefile 会自动拉取；安装后若终端未识别 `uv`，请执行：

```bash
source $HOME/.local/bin/env
```

再执行一次 `make install`。

### 2. 仅启动后端

在 **pm-copilot** 目录下：

```bash
make start-backend
```

- 后端地址：**http://127.0.0.1:8001**（默认，可通过环境变量覆盖）
- 可用于：单独调试 API、用 curl/Postman 测接口、或先起后端再起前端。

### 3. 仅启动前端

在 **pm-copilot** 目录下：

```bash
make start-frontend
```

- 前端地址：**http://localhost:5173**（Vite 默认）
- 前端会通过代理把 `/api` 请求转发到后端；**请先启动后端**，否则页面会提示 “Backend Unavailable”。
- 若后端不在本机或端口不同，在 `pm-copilot/frontend` 下新建或修改 `.env.development`，设置：
  - `VITE_API_BASE=http://<后端地址>:<端口>`  
  例如：`VITE_API_BASE=http://192.168.1.100:8001`

### 4. 同时启动前后端

在 **pm-copilot** 目录下：

```bash
make dev
```

会同时启动后端（默认 8001）和前端（5173），适合一次性跑完整应用。

### 5. 在项目根目录执行

在 **Copilot** 根目录也可委托给 pm-copilot：

```bash
make -C pm-copilot install    # 安装
make -C pm-copilot start-backend   # 仅后端
make -C pm-copilot start-frontend  # 仅前端
make -C pm-copilot dev         # 前后端一起
```

或使用根目录的 Makefile（已委托到 pm-copilot）：

```bash
make start-backend
make start-frontend
make dev
```

### 6. 后端环境变量

后端配置在 **pm-copilot/app/.env**，需至少配置：

- `LLM_MODEL`、`LLM_MODEL_API_KEY`、`LLM_BASE_URL`（DeepSeek / LiteLLM）
- `TAVILY_API_KEY`（Tavily 搜索）

可选启动覆盖（临时改端口/主机）：

```bash
BACKEND_PORT=8010 make start-backend
BACKEND_PORT=8010 make start-frontend
```
